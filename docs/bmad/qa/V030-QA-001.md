## Test cases (22 new)

### processIdea — happy path (single mode) [4 tests]
- Calls all steps in sequence, marks idea done
- Schedules one post per image in single mode
- Falls back to expandedPrompt when `generatePostContent` returns `undefined` (pi-down)
- Falls back to expandedPrompt when `generatePostContent` throws (pi-down)

### processIdea — image generation timeout [1 test]
- `waitForImages([])` → logs "Timed out", skips caption/schedule, marks done

### processIdea — skip signal [3 tests]
- `isSkipRequested=true` → throws `SkipIdeaSignal` at first `checkSkip`
- `waitForImages` throws `SkipIdeaSignal` → propagates correctly
- `SkipIdeaSignal` not swallowed by carousel caption `try/catch`

### processIdea — carousel mode [2 tests]
- `generatePostContent` called once (anchor only); `saveImage` called 4× (3 raw + 1 captioned)
- Falls back to expandedPrompt when carousel caption returns `undefined`

### processIdea — pipeline stage toggles [3 tests]
- `autoCaption=false` → `generatePostContent` not called
- `autoSchedule=false` → `findNextAvailableSlot` not called
- No platforms → `addLog('schedule', ..., 'error', 'No platforms...')`, `findNextAvailableSlot` not called

### processIdea — recoverable errors [3 tests]
- `fetchTrendingContext` throws → logs error, continues with empty context, reaches `done`
- `expandIdeaToPrompt` throws → propagates (fatal), idea never marked done
- `triggerImageGeneration` throws → propagates (fatal), idea never marked done

### processIdea — accumulatedPosts mutation [1 test]
- Two images scheduled → 2 entries appended to `accumulatedPosts` with correct `sourceIdeaId`

### withPipelineRunning [5 tests]
- Sets `running(true)` then `running(false)` on success
- Returns fn's return value
- Sets `running(false)` even when fn throws (the fix)
- Sets `running(false)` even when fn throws `SkipIdeaSignal`
- `running(false)` fires **before** the error propagates to the caller (order verified)
