### SEC-001 — SSRF allowlist in `app/api/proxy-image/route.ts`

- [INFO] `isAllowedUrl()` exported as a pure function — directly testable and tested. ✓
- [INFO] Allowlist: `cdn.leonardo.ai` (exact), `i.uguu.se` (exact),
  `*.storage.googleapis.com` (suffix). These are the three actual upstream image sources. ✓
- [INFO] `parsed.protocol !== 'https:'` check — blocks `http://`, `file://`, `gopher://`,
  and every other non-HTTPS scheme before host inspection. ✓
- [INFO] Suffix lookalike bypass correctly prevented: suffix list uses `.storage.googleapis.com`
  (leading dot required), so `cdn.leonardo.ai.evil.com` is rejected by the exact-host check,
  not the suffix list. ✓
- [INFO] Returns 403 (not 400) — correct semantics: "refused" vs "malformed request". ✓
- [INFO] 14 vitest tests cover all meaningful bypass vectors including bare
  `storage.googleapis.com` (no subdomain → rejected), localhost, 127.0.0.1,
  case-insensitive hostname, empty string, malformed URL. Comprehensive. ✓

### SEC-003 — POSIX shell injection fix in `pi/setup`

- [INFO] `execSync(template_string)` → `spawnSync('tmux', [...args])` array form.
  `piPath` is now a single argv element passed directly to `execve`, not interpolated
  into a shell-parsed string. Any character valid in a filesystem path is safe. ✓
- [INFO] `tmux kill-session` also converted to `spawnSync` array form. Consistent. ✓
- [INFO] Exit code check added: `if (tmuxResult.status !== 0) throw new Error(...)`.
  Prior `execSync` threw on non-zero; `spawnSync` does not — the explicit check restores
  parity. ✓
- [INFO] Windows branch correctly left alone — Win32 reserves `"` in paths, so the
  double-quote fence cannot be broken by a valid filesystem path. ✓
