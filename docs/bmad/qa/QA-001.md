### Issues in STORY-001 review work (not in the review artifact itself)

**WARN — Missing `permissions: contents: write` in workflow**

`softprops/action-gh-release@v2` uses `GITHUB_TOKEN` to create a draft release
but no `permissions:` block exists at the job level. Repos with restricted
default token permissions will 403 on the first tag push. This only affects
the draft-release step; main-branch builds are unaffected.

Developer flagged this as a known risk but did not fix it preemptively.
Recommend adding before tagging any release:
```yaml
jobs:
  build:
    permissions:
      contents: write
```

**INFO — Hardcoded Node cache key**

`key: bundled-node-v22.11.0-${{ runner.os }}` (line 58 of workflow) will
drift silently if `scripts/fetch-windows-node.ps1` bumps its pinned version
without a corresponding YAML edit. Low risk short-term; note for next Node bump.

> Description: Configure tauri.conf.json for Windows  
> Acceptance: App opens with Vercel URL in native webview

### Code audit — what actually shipped (commit `fbf81a5`)

Reviewed `src-tauri/tauri.conf.json` and `src-tauri/src/lib.rs` directly.

**`tauri.conf.json`:**
- `frontendDist: "../src-tauri/frontend-stub"` — local loading screen, no Vercel URL
- `app.windows[0].url: "index.html"` — loads stub at launch
- `bundle.resources: ["resources/**/*"]` — bundles Node + Next standalone + pi

**`src-tauri/src/lib.rs` — Rust launcher:**
- Resolves bundled `node.exe` from `resources/node/node.exe`
- Resolves bundled pi from `resources/pi/pi.cmd`
- Picks a free ephemeral TCP port via `TcpListener::bind("127.0.0.1:0")`
- Spawns `node start.js` with `PORT`, `HOSTNAME=127.0.0.1`, `MASHUPFORGE_RESOURCES_DIR`, `PI_BIN` set
- Polls the port for up to 30s; on success calls `window.navigate(http://127.0.0.1:{port})`
- On `CloseRequested` kills the sidecar child process

**There is no Vercel URL anywhere in the config or Rust code.**

### Verdict: STORY-002 acceptance criteria are obsolete AND superseded

The accepted architecture diverged from the brief's "Use Vercel URL as webview
source (thin installer)" constraint before STORY-002 was ever implemented.
Commit `fbf81a5` delivered a substantially more capable implementation:

| Brief intent | What shipped |
|---|---|
| Vercel URL in webview | Bundled Node sidecar on 127.0.0.1:{ephemeral-port} |
| Thin installer | Full standalone: Node + Next build + pi baked into resources/ |
| Internet dependency for AI | No internet required — pi runs locally |

The real acceptance criteria that were satisfied by `fbf81a5`:
- `tauri.conf.json` configured for Windows ✓
- App launches and shows a loading screen ✓
- Rust launcher spawns bundled Node on a random 127.0.0.1 port ✓
- Window navigates to local URL once server accepts connections ✓
- Sidecar killed cleanly on window close ✓
- PI_BIN wired so `/api/pi/start` can find the bundled binary ✓

**Recommendation:** Close STORY-002 as *satisfied by commit `fbf81a5`* with
rewritten acceptance criteria (see Developer's STORY-001 review, §Observation 1).
No new work required.
