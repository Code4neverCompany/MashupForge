# STORY: V080-DEV-001 — Gallery Batch Checkbox Fix

## Status: Done
## Assignee: developer
## Brief: V080-POLISH

## Summary
Gallery batch checkboxes in GalleryCard.tsx are unclickable. The checkbox at line 214 renders with `selectedForBatch` / `onToggleBatch` but clicking doesn't toggle.

## Acceptance Criteria
- [x] Clicking the checkbox toggles the image in/out of `selectedForBatch`
- [x] Visual state updates immediately (checked/unchecked)
- [x] Batch actions (delete, tag, etc.) work with the toggled selection
- [x] No regression on card click, context menu, or other interactive elements

## Resolution
Fix landed with the z-40 bump on the checkbox wrapper (`components/GalleryCard.tsx`
line 229) + `e.stopPropagation()` on `onClick` / `onChange`. The top action
overlay at z-30 was eating the hit test on the empty left half of the card
because it rendered later in the DOM at equal z. Bumping the checkbox
wrapper to z-40 pins it above. Covered by 7 tests in
`tests/components/GalleryCard.test.tsx` (render / non-render / toggle
add+remove / no-propagate / z-class pin) — all passing.

## Technical Notes
- GalleryCard.tsx lines 214-221: checkbox with onChange handler
- The card likely has an onClick that swallows the checkbox event
- Check for `e.stopPropagation()` on the checkbox or parent elements
- Check z-index: the checkbox may be behind another element
- The `selectedForBatch` Set is passed as prop — verify it's not being recreated each render (which would break Set.has())
