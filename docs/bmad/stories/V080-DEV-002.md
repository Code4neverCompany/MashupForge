# STORY: V080-DEV-002 — Disapproved Pipeline Images in Gallery

## Status: Done
## Assignee: developer
## Brief: V080-POLISH

## Summary
Images that were disapproved/rejected in the Pipeline approval queue still appear in the Gallery tab. The current filter only checks `pipelinePending === true` but rejected images may lose that flag.

## Acceptance Criteria
- [x] Images with pipeline status 'rejected' do NOT appear in Gallery
- [x] Images whose ScheduledPost status is 'rejected' do NOT appear in Gallery
- [x] Images that are genuinely finalized (approved + posted) DO appear in Gallery
- [x] Pipeline tab still shows rejected images for reference

## Resolution
Added `lib/gallery-visibility.ts::getAllRejectedImageIds(scheduledPosts)`
— a pure O(N) pass that returns the set of imageIds whose ScheduledPosts
ALL have status `'rejected'`. `MainContent.tsx::displayedImages` filters
the grid with `allRejectedImageIds.has(img.id)`. An image with at least
one non-rejected sibling post (pending_approval / scheduled / posted /
failed) stays visible, so partial carousel-sibling rejections don't
disappear the image; images with no posts at all (manually-generated,
never-scheduled) are also unaffected. Covered by
`tests/lib/gallery-visibility.test.ts`.
