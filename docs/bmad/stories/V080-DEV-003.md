# STORY: V080-DEV-003 — Pipeline 2-Image Carousel Deselection

## Status: Open
## Assignee: developer
## Brief: V080-POLISH

## Summary
When a carousel has exactly 2 images, the user cannot deselect either one because `canRejectMoreInCarousel` requires `nonRejectedCount > CAROUSEL_MIN_IMAGES (2)`. This locks the user — they can't reduce to 1 image.

## Acceptance Criteria
- [ ] User CAN reject one image from a 2-image carousel
- [ ] When reduced to 1 image, the carousel degrades to a single post (not a broken 1-image carousel)
- [ ] The single remaining image is posted as a standalone image post
- [ ] No error prompts or broken UI states
- [ ] Whole-carousel reject still works (rejects all images)

## Technical Notes
- lib/carousel-degrade-guard.ts: `CAROUSEL_MIN_IMAGES = 2`, `canRejectMoreInCarousel` returns `nonRejectedCount > 2`
- When user rejects 1 of 2 images → remaining image should be converted to single post
- The auto-poster rejects 1-image carousels (per the comment in degrade-guard)
- Solution: allow the rejection, then handle the degrade — either:
  a) Convert the ScheduledPost from carousel to single image, or
  b) Show a confirmation: "This will post as a single image instead of a carousel"
- Check CarouselApprovalCard.tsx line 177 for the reject handler
- Check lib/carousel-ghost.ts for ghost/degrade logic
