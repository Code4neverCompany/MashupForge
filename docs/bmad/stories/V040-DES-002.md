# V040-DES-002 — Platform health dot + daily digest card

**Status:** spec
**Author:** Designer
**Classification:** routine (both components are single-file, presentational, <50 LOC each)
**Scope:** two small UX wins from the v0.3.0 debrief — surface credential health continuously, and give the Ideas tab a morning briefing the user can glance at before doing anything else

---

## Part A — Platform credential health dot

### A.0 · Problem

Right now the only way to know if Instagram/Pinterest/Twitter/Discord credentials are working is to:
1. Open Settings modal
2. Click through each platform section
3. Read individual status text

So the user discovers broken credentials **at post time** — when a scheduled post silently fails or the pipeline stalls. Unacceptable for a tool whose job is automation.

Need: a persistent, always-visible signal that tells the user "your platforms are (or aren't) working" without leaving their current view.

### A.1 · Where it lives

**Primary placement: bottom of `components/Sidebar.tsx`** — a single-line persistent status strip anchored to the bottom of the AI chat panel, above the tab switcher footer.

Rationale:
- Always visible whenever the sidebar is open (which is almost always — Sidebar hosts the content-generator, the primary input path)
- Doesn't compete with the view tab bar (which stays focused on navigation)
- Sits near other persistent chrome (settings access) so the cognitive pairing "status + settings" is tight

**When sidebar is collapsed** (`!isSidebarOpen`): render a tighter 4-dot rail inside the collapsed-sidebar icon column. Same colors, no labels, same tooltips on hover.

### A.2 · Layout (sidebar-open state)

```
┌───────────────────────────────────────────────────────┐
│                                                       │
│            (AI chat / content generator)              │
│                                                       │
├───────────────────────────────────────────────────────┤
│  ● IG   ● PN   ● TW   ● DC        [⚙ Settings]       │   ← new strip
└───────────────────────────────────────────────────────┘
```

- Horizontal flex row, `px-3 py-2 border-t border-zinc-800/70 bg-zinc-950/60`
- Each platform = one `<HealthDot platform="..." />` button (see A.4)
- Settings gear on the right is pre-existing; health strip sits left of it
- If user has 0 platforms configured, hide the strip entirely (no "empty state" — don't waste pixels)

### A.3 · Four health states

| State | Color | Dot class | When |
|---|---|---|---|
| Healthy | emerald-500 | `bg-emerald-500 shadow-[0_0_4px_rgba(16,185,129,0.6)]` | creds present + last successful post/check ≤24h ago |
| Stale | amber-400 | `bg-amber-400` | creds present + last check >24h ago or never checked |
| Broken | red-500 | `bg-red-500 animate-pulse` | creds missing, invalid, or last action failed with auth/401 |
| Unused | zinc-600 | `bg-zinc-600/70` | platform not in `settings.pipelinePlatforms` AND no scheduled posts use it |

Pulsing is reserved for **broken only** — it's an attention demand and should be rare. Healthy/stale/unused stay still.

Reuses the calendar status palette language (`MainContent.tsx:455-460`): emerald = success, amber = warning, red = failure, zinc = neutral.

### A.4 · Dot component anatomy

```
┌──────────────┐
│   ●  IG      │   ← dot (8px) + 2-letter platform code
└──────────────┘
       ↓ hover
  ┌───────────────────────────────────┐
  │ Instagram · Healthy               │
  │ Last post: 2 hr ago · 3 in queue  │
  │ [Open settings]                   │
  └───────────────────────────────────┘
```

Visual spec:
- Button: `inline-flex items-center gap-1.5 px-2 py-1 rounded-lg hover:bg-zinc-800/60 transition-colors`
- Dot: `w-2 h-2 rounded-full` + state class from A.3
- Label: 2-letter code in `text-[10px] font-semibold text-zinc-400`, grouped mono-ish but uses normal AETHER font (no new typeface)
- Platform codes: `IG` · `PN` · `TW` · `DC` (uppercase, fixed)

Tooltip (on hover, 300ms delay; on focus-visible, instant):
- Line 1: `{Platform name} · {state label}` — state label = "Healthy" / "Stale" / "Broken" / "Unused"
- Line 2: one-line context
  - Healthy: "Last post: {relative} · {n} in queue"
  - Stale: "Last check: {relative}. Credentials may have expired."
  - Broken: the specific error — "Missing access token" / "Auth failed (401)" / "Rate-limited"
  - Unused: "Not in pipeline. [Enable in settings]"
- Line 3: `[Open settings]` link — scrolls Settings modal to that platform's section

Click behavior:
- Healthy/Stale: opens Settings modal scrolled to platform section
- Broken: opens Settings modal AND opens the specific failure banner for that platform
- Unused: opens Settings modal to pipeline-platforms config

### A.5 · A11y

- `<button aria-label="Instagram · Healthy · click for settings">` — label is computed, not static
- `role="status"` on the strip container so screen readers announce changes
- State changes announce via `aria-live="polite"` region parented to the strip (broken state coming online should NOT interrupt, just queue)
- `prefers-reduced-motion`: broken-state pulse becomes static red with a `⚠` badge instead

### A.6 · Data source

All four states are derivable from existing data — **no new schema required** to ship the visual:

| Read | From |
|---|---|
| Credentials present? | `settings.apiKeys.{platform}` (existing, `types/mashup.ts:167-184`) |
| Last post timestamp | `settings.scheduledPosts.filter(p => p.status === 'posted').map(p => p.date+p.time).max()` |
| Queue depth | `settings.scheduledPosts.filter(p => p.platforms.includes(platform) && p.status === 'scheduled').length` |
| Last failure? | `settings.scheduledPosts.filter(p => p.status === 'failed' && p.platforms.includes(platform))[-1]` |

Note: this gets us to ~80% correct. The remaining 20% (active token validation ping) is a **separate future PROP** — health dot gracefully degrades to "Stale" when it can't confirm, which is honest.

### A.7 · Routing

**Routine — Designer self-implements:**
- `components/platform/HealthDot.tsx` (single file, <50 LOC)
- `components/platform/HealthStrip.tsx` (single file, <50 LOC)
- Mount call site in `Sidebar.tsx` (single-line JSX addition)
- Derivation helpers live inline in HealthStrip; no new lib file

**Complex — needs PROP:**
- Active credential-validation worker (e.g., periodic `/api/{platform}/health` ping)
- Persisted `lastCheckedAt` / `lastCheckResult` on platform creds
- Any `settings` schema change

## Part B — Daily digest card (Ideas tab)

### B.0 · Problem

Ideas tab currently opens to three empty kanban columns. The user has to:
- Switch to Pipeline to see automation status
- Switch to Calendar to see week-fill
- Switch to Approval to see pending work
- Switch to Post Ready to see what shipped

The Ideas tab is where the user's brainstorming session starts — they open it to think, not to navigate. Give them a 5-second glance at "where we are today" so they can decide if they should brainstorm more or let existing work ship.

### B.1 · Where it lives

**Above the kanban columns, below the Ideas Board header.** Insertion point: `components/MainContent.tsx:1908` — between the closing `</div>` of the section header and the opening `<div>` of the kanban columns at L1910.

Collapsible with a small `[Hide digest]` button in the top-right corner; state persists to `localStorage` under `mashup.ideas.digestHidden`. If the user hides it, they can re-open via a tiny `[Show digest ▾]` button in the header row.

### B.2 · Card anatomy

```
┌──────────────────────────────────────────────────────────────────┐
│ ☀ Good morning · Sat Apr 18                    [Hide digest ✕]   │
│                                                                  │
│ ┌──────────────┬──────────────┬──────────────┬──────────────┐    │
│ │  Yesterday   │   Week fill  │   Pending    │   Pipeline   │    │
│ │              │              │              │              │    │
│ │    3 posts   │   9 / 14     │   2 waiting  │   [●]        │    │
│ │    shipped   │   slots      │   approval   │   Running    │    │
│ │              │   [====-----]│   [Review ▸] │              │    │
│ └──────────────┴──────────────┴──────────────┴──────────────┘    │
│                                                                  │
│  Next post · Sat 18:00 · Instagram + Pinterest                   │
│  Platforms: ● IG  ● PN  ● TW  ⚠ DC                               │
└──────────────────────────────────────────────────────────────────┘
```

- Container: `bg-zinc-900/40 border border-[#c5a062]/20 rounded-2xl p-5 space-y-4`
- Greeting line: `type-title`-ish (but smaller: `text-base font-semibold`), icon varies by time-of-day:
  - 5am–11am: `Sunrise` "Good morning"
  - 11am–5pm: `Sun` "Good afternoon"
  - 5pm–10pm: `Sunset` "Good evening"
  - 10pm–5am: `Moon` "Still up?"
  - Keeps brand warmth without getting cute — no emojis
- Hide button: `text-xs text-zinc-500 hover:text-zinc-300`

### B.3 · Four metric tiles

Each tile: `bg-zinc-950/60 rounded-xl p-3 space-y-1 text-center`

| Tile | Title | Primary | Secondary |
|---|---|---|---|
| Yesterday | uppercase zinc-500 10px | `3 posts shipped` — large number + small label | "+2 from day before" (green/red trend) |
| Week fill | "Week fill" | `9 / 14 slots` | mini progress bar (§B.4) |
| Pending | "Pending" | `2 waiting approval` | `[Review ▸]` button → `setView('approval')` |
| Pipeline | "Pipeline" | state pill (see §B.5) | last-event timestamp: "last: 4 min ago" |

Empty states per tile:
- 0 posts shipped yesterday: "0 posts · pipeline paused?" in zinc-500 (not red — yesterday being empty is not alarming)
- 0 pending approvals: "All clear" in emerald-400/80
- 0 week-fill slots: "Empty week" + small `[Auto-fill]` button (V030-DES-001 §B hook)
- Pipeline stopped: "Paused" pill in amber-500/20

### B.4 · Week-fill mini bar

Reuses V030-DES-001 §B palette:
- Track: `bg-zinc-800 h-1.5 rounded-full`
- Fill: `bg-[#c5a062]` for `< 70%`, `bg-amber-500` for `70–100%`, `bg-emerald-500` for `100%`
- Width: `${(filled/target)*100}%`
- No text inside the bar — the `9/14` above is enough

### B.5 · Pipeline state pill

| State | Pill |
|---|---|
| Running + processing | `bg-emerald-500/20 text-emerald-300 border border-emerald-500/40` + pulsing green dot · "Running" |
| Running but idle | `bg-zinc-800 text-zinc-300 border border-zinc-700` + steady zinc dot · "Idle" |
| Paused (enabled, not running) | `bg-amber-500/20 text-amber-300 border border-amber-500/40` · "Paused" |
| Disabled | `bg-zinc-900 text-zinc-500 border border-zinc-800` · "Off" |

Clicking the pill opens Pipeline view — provides a fast pivot from "idea triage" back to "automation oversight."

### B.6 · Footer row

Below the 4-tile grid:
- "Next post" line — shows the nearest scheduled post: date/time + platform chips. If no next post: "No posts scheduled" in zinc-500.
- Platform status mini-row — same HealthDot components from Part A, no labels, smaller (1.5px dots). Provides a second touchpoint for health without forcing the sidebar to be open.

### B.7 · Responsive

- Desktop (≥md): 4 tiles in a row, full footer row
- Tablet (sm–md): 2x2 tile grid, footer row wraps
- Mobile (<sm): 1 column stack — tiles and footer rows all vertical; greeting stays at top

Tiles themselves don't resize — the grid does.

### B.8 · A11y + motion

- `role="region" aria-label="Daily digest"` on the container
- Hide button has `aria-expanded` + controls the region
- Trend arrow icons are decorative (`aria-hidden="true"`); trend is also in text ("+2 from day before")
- Pipeline state pulse respects `prefers-reduced-motion`
- No carousel, no auto-refresh animation — the whole card is static until the user reloads the view. Keeps the Ideas tab calm for brainstorming.

### B.9 · Data sources (all existing)

| Field | From |
|---|---|
| Posts shipped yesterday | `scheduledPosts.filter(p => p.status === 'posted' && isYesterday(p.date))` |
| Day-before comparison | same filter, shifted one day |
| Week fill filled/target | existing `weekFillStatus` from context + `settings.pipelineTargetDays` |
| Pending approvals | `scheduledPosts.filter(p => p.status === 'pending_approval').length` |
| Pipeline state | `pipelineEnabled + pipelineRunning + pipelineQueue.length` from context |
| Last pipeline event | `pipelineLog[0]?.timestamp` |
| Next scheduled post | `scheduledPosts.filter(p => p.status === 'scheduled').sort(by date+time)[0]` |
| Platform health | derivations from Part A, reused |

Zero new schema. Zero new context methods.

### B.10 · Routing

**Routine — Designer self-implements:**
- `components/ideas/DailyDigest.tsx` (single file, ~150 LOC)
- `components/ideas/DigestTile.tsx` (~40 LOC, reusable for all 4 tiles)
- Mount call in `MainContent.tsx` (single-line JSX)
- `useLocalStorage` for hide/show state (inline hook, no new util)

**Complex — PROP needed:**
- Nothing. All data sources exist. This ships as-is.

## Cross-cutting decisions

### Colors
Health dot and digest tile colors use the same calendar palette:
- emerald = good/shipped
- amber = warning/paused
- red = broken/failed
- gold (`#c5a062`) = pipeline interactive accent / progress fill
- indigo = reserved for approval queue (V030-DES-003)
- zinc = neutral/inactive

Consistent language across the whole app.

### Iconography
- Health dot: no icon — the dot is the icon
- Digest tile headers: no per-tile icons; the label + number is enough
- Greeting: weather-time icon only (Sunrise/Sun/Sunset/Moon)
- Pipeline pill: pulsing dot only

**Resisted temptation** to icon-crowd the digest card. It's a dashboard, not a toolbar — whitespace is the design.

### Typography
- Tile primary number: `text-2xl font-semibold`
- Tile label: `text-[10px] uppercase tracking-wide text-zinc-500`
- Tile secondary: `text-xs text-zinc-400`
- Dot label: `text-[10px] font-semibold text-zinc-400`

### Interaction priorities
Digest order from left to right is **retrospective → prospective**:
1. Yesterday (looking back)
2. Week fill (current state)
3. Pending (near-term action)
4. Pipeline (ongoing process)

Mirrors how a user thinks in the morning: "what shipped? where are we? what do I need to do? is the machine running?"
