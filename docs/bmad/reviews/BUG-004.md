# BUG-004 — Tauri v2 ACL rejects `plugin:updater|check` on Windows

**Status:** done · **Branch:** main · **Classification:** routine

## TL;DR
User report on Windows: `"Command plugin:updater|check not allowed by ACL"` at launch-time update check. Root cause: `src-tauri/capabilities/default.json` listed `updater:default` + `updater:allow-check` + `updater:allow-download-and-install`, but the explicit `allow-download` and `allow-install` permissions were missing. Even though `updater:default` nominally expands to include all four commands (verified against `tauri-plugin-updater-2.10.1/permissions/default.toml`), a shipped build was failing the ACL check on `check` — belt-and-suspenders fix is to enumerate every updater command explicitly.

## Root cause
`tauri-plugin-updater` 2.10.1 exports four command permissions:
- `allow-check` → `check` command
- `allow-download` → `download` command
- `allow-install` → `install` command
- `allow-download-and-install` → `downloadAndInstall` command

Only two were explicit in capabilities: `allow-check` and `allow-download-and-install`. The `updater:default` preset should have covered the other two, but the runtime error the user reported targets `check` — which *was* explicitly listed. That suggests either:
(a) the installed binary was built against a different capability snapshot, or
(b) the ACL resolver isn't expanding `updater:default` the way we expect.

Either way, the safe fix is to list every updater command explicitly. That removes the dependency on preset-alias resolution.

## Fix
`src-tauri/capabilities/default.json`:

```diff
 "updater:default",
 "updater:allow-check",
+"updater:allow-download",
+"updater:allow-install",
 "updater:allow-download-and-install",
```

`updater:default` is kept (still acts as a sanity fallback).

Other plugin permissions audited at the same time:
- `process:default` + `process:allow-restart` — covers `relaunch()` (BUG-002). Verified against `tauri-plugin-process-2.3.1` permissions (exports `allow-exit` + `allow-restart`; `relaunch()` JS → `restart` Rust command). Already correct.
- `opener:default` + `opener:allow-open-url` — already correct.
- `autostart:*` — already explicit on all three commands.
