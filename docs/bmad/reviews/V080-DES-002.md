# Review — V080-DES-002 Post Ready Countdown Timer

**TLDR:** The bulk of this story was already shipped — `components/postready/CountdownBadge.tsx` exists with a 60s `setInterval`, three-tone styling (gold/amber/red), the full set of label formats from the AC ("in 2h 15m", "in 1d 3h", "overdue by 10m"), and is rendered in both `PostReadyCard` and `PostReadyCarouselCard` whenever `kind === 'scheduled'`. The one missing AC was the unscheduled-state affordance — when a card is in Post Ready but no `scheduledPost` exists yet, nothing rendered, which made an unscheduled card visually indistinguishable from a "scheduled but no time set" card. Added a neutral `Clock + "Not scheduled"` pill alongside the countdown slot for `kind === 'ready'` in both cards.

## Files changed
- `components/postready/PostReadyCard.tsx` — sibling pill for the unscheduled case.
- `components/postready/PostReadyCarouselCard.tsx` — same treatment for carousels.

## Acceptance criteria
- [x] Each PostReady card shows a countdown badge — already wired (`CountdownBadge` rendered when `kind === 'scheduled'`).
- [x] Countdown updates in real-time (or at least every minute) — `setInterval(..., 60_000)` in `CountdownBadge.tsx:60`.
- [x] Overdue posts are visually distinct (red/amber badge) — `tone === 'overdue'` → red pill; `tone === 'soon'` (<1h) → amber; otherwise gold.
- [x] Unscheduled posts show "Not scheduled" or similar — **NEW**: neutral pill with Clock icon, only for `kind === 'ready'` so posted/failed cards don't double up on status indicators.
- [x] Badge doesn't clutter the card — single-line, `px-2 py-0.5 text-[10px]`, sits in the existing top status-pill row.

## What I deliberately did NOT change
- **CountdownBadge internals.** Already pinned by `tests/components/CountdownBadge.toTimestamp.test.tsx` (V081-TEST-GAPS) — empty-time NaN-slip is covered, label formatting is correct.
- **Posted / failed states.** The status pill already says enough; adding a redundant timer would be noise.
- **Refresh cadence.** 60s is right per the AC ("at least every minute") and avoids re-render storms when a Post Ready tab has 50+ cards.
