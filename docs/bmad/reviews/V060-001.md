# V060-001 — Post Ready card redesign

**Status:** done
**Branch:** main
**Tests:** 456 → 467 (+11)

## Why

The Post Ready cards were overwhelming: caption + hashtag editor, full
date/time picker, six secondary buttons, status badge buried in the
image overlay. Schedule was technically a primary action but it competed
visually with five neighbors. Maurice approved a simplified card with a
schedule-first layout.

## What changed

New components under `components/postready/`:

- **`InlineScheduleCalendar.tsx`** — 14-day grid (heatmap ON by default
  per AC-6), top-3 day stars from average engagement score, drill-down
  time picker with the day's top 3 hours highlighted, plus an
  Auto-Schedule shortcut that calls `findBestSlot` against every
  scheduled post. Reuses `HeatmapTint` / `HeatmapToggleButton` from
  `WeekHeatmap` and `computeWeekScores` from `smartScheduler`.
- **`PostReadyCard.tsx`** — single-image card. Top: status pill +
  colored 2px border (Ready/Scheduled/Posted/Failed). Body: aspect-
  cropped image, 2-line caption preview (tap to edit), 3 + N more
  hashtag chips, platform chips. Footer: **Post Now** + **Schedule** +
  kebab menu. Kebab hides Copy / Regen / Move out of Post Ready until
  opened. Schedule button toggles the inline calendar below the card.
- **`PostReadyCarouselCard.tsx`** — same layout for multi-image
  carousels. Image strip across the top instead of `AspectPreview`.
  **Separate** / **Lock as group** moved into the kebab (AC-11).
- **`lib/post-ready-status.ts`** — single `derivePostReadyStatus`
  helper used by both cards. Replaces the duplicated badge ternaries
  in `MainContent.tsx` and pins the BUG-CRIT-007 contract.

`components/MainContent.tsx`: replaced ~520 lines of inline JSX with
two component calls inside the existing `computeCarouselView(ready).map`.

## Tests

- `tests/integration/post-ready-card.test.tsx` (new, 11 cases) covers
  every acceptance criterion that has a JSX-observable surface: status
  pill kind for each lifecycle state, caption collapse + expand,
  hashtag 3 + N more pattern, kebab visibility, calendar toggle,
  Auto-Schedule confirms + closes calendar.
- `tests/integration/carousel-badge-derivation.test.ts` repointed at
  the new shared helper. The previous "no badge for `pending_approval`"
  expectation is updated: the redesign surfaces pending as a
  scheduled-class status so the colored border + pill make the
  lifecycle visible (a deliberate UX improvement, not a regression).

## Notes

- The card's `busy` prop union includes `null` to match the
  `Record<string, 'posting' | 'scheduling' | null>` shape from
  `MainContent.postBusy` — no cast at the call site.
- `derivePostReadyStatus` keeps the manual-Post-Now path winning
  over `ScheduledPost.status` (BUG-CRIT-007 contract preserved).
- Date/time inline inputs were removed from the card — the inline
  calendar is now the only schedule path. Auto-Schedule is the
  fast-track; manual day+time picking is the explicit path.
