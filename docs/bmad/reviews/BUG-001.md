# BUG-001 ‚Äî Batch Caption now treats carousels as one unit

**Why:** Carousels should have ONE shared caption, not one per image ‚Äî overkill and wrong UX.
**Classification:** routine
**Executed:** 2026-04-18 (developer subagent)

## Root cause

`batchCaptionImages` in `components/MainContent.tsx` (formerly line 746) iterated the **flat** `visible` list and called `generatePostContent` once per uncaptioned image ‚Äî burning N AI calls per carousel and producing N different captions for what UX-wise is a single post.

The Captioning tab's per-card "Generate" button on a carousel (line 2430-2447) already did the right thing: caption the anchor once, then `patchImage` the rest of the group with the same `postCaption` + `postHashtags`. The auto-pipeline (`hooks/usePipeline.ts:312-335`) follows the same "shared caption" pattern.

`batchCaptionImages` was the only path that hadn't been carousel-ified.

## Fix

Single-file change to `components/MainContent.tsx` (+54 / ‚àí8).

`batchCaptionImages` now:

1. **Bucket inputs** ‚Äî when `captioningGrouped` is on, run `computeCarouselView(candidates)` and produce one `Entry` per carousel + one per single. When grouped is off, behave exactly as before (one entry per uncaptioned image).
2. **Per carousel entry** ‚Äî anchor = first image. If anchor already has a caption, just `patchImage` the rest with the anchor's existing caption (no AI call). Otherwise call `generatePostContent(anchor)` once and fan its result out to every other image in the group via `patchImage`.
3. **Progress meter** ‚Äî `total` now counts carousel groups + singles, not raw images. The "N/N" the user sees matches the number of distinct posts, which is what they'd intuitively expect.
4. **Single images** ‚Äî unchanged: one `generatePostContent` call each.

Mirrors the per-card Generate button behavior exactly, so the two entry points stay in sync.

1. Open Captioning tab with several uncaptioned images, some clearly grouped within the 5-minute carousel auto-window (or via explicit `carouselGroups`).

3. Click **Batch Caption**.
4. **Expect:** progress shows `N/N` where N = (singles + distinct carousels), not raw image count. Each carousel ends with all images sharing the same `postCaption` + `postHashtags`. Each single image has its own caption.
