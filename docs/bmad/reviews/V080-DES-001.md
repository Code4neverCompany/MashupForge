# Review — V080-DES-001 Onboarding Scroll & Tag Expansion

**TLDR:** Two surgical fixes. (1) The onboarding dialog's `min-h-[540px]` was an unconditional floor — on phones taller than 360px but shorter than ~580px, the modal stretched past the viewport and the Continue button slid below the fold. Gated to `sm:min-h-[540px]` so the floor only applies at the 640px+ breakpoint where it makes sense; smaller viewports rely on the body's existing `flex-1 overflow-y-auto` pattern. (2) Tag pool expanded from 28 universes / 20 genres → 32 / 32 to clear the "30+" target each side. The 10-item selection cap is unchanged — it's pinned by `tests/components/Step2Niche-cap.test.tsx` (V081-TEST-GAPS) and "10+ tags" reads as "at least 10" on a fresh read.

## Files changed
- `components/onboarding/OnboardingWizard.tsx` — drop unconditional `min-h-[540px]`.
- `components/onboarding/steps/Step2Niche.tsx` — pool +4 universes / +12 genres.

## Acceptance criteria
- [x] Continue/Next button is ALWAYS visible and clickable regardless of content height — fixed by removing the small-screen min-h floor; flex-1+overflow-y-auto on the body and flex-shrink-0 on the footer were already correct.
- [x] The scrollable content area does not push buttons out of view — same fix.
- [x] User can select 10+ tags (no artificial low limit) — already met (cap = 10, pinned by V081-TEST-GAPS test).
- [x] Tag pool expanded to 30+ recommended tags across niches and genres — universes 28→32, genres 20→32.
- [x] Selected tags are visually distinct from unselected — already correct (`border-[#c5a062] bg-[#c5a062]/15 text-[#c5a062]` on selected vs `border-zinc-700 text-zinc-300` on unselected).
- [x] Scroll works smoothly within the tag area — body owns the scroll, no inner overflow constraints introduced.

## What I deliberately did NOT change
- **MAX_SELECTIONS = 10.** The story narrative ("Only ~2 tags can be selected") is stale — the cap was raised 2 → 10 in the original V080-DES-001 pass and pinned by V081-TEST-GAPS. Bumping it would have churned the test contract for unclear product benefit. If the user actually wants > 10, re-open with an explicit number.
- **Step1 / Step3 / Stepper layout.** Out of scope.
