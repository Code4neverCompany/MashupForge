# V040-005 — Morning-briefing surface above the Ideas kanban

Implements Part B of `docs/bmad/stories/V040-DES-002.md`. Part A
(platform health dot in the sidebar) is a separate follow-up —
explicitly out of scope for this task and untouched.

## Why

The Ideas tab currently opens to three empty kanban columns. To get
situational awareness the user has to rotate through Pipeline (for
daemon state), Calendar (for week-fill), Approval queue in Pipeline
(for pending work), and Post Ready (for shipped). That's four tab
switches for a 5-second "where are we today?" glance.

Ideas is where brainstorming starts. The digest gives a morning
briefing so the user can immediately decide whether to brainstorm
more or let existing work ship.

## What changed

### `components/ideas/DigestTile.tsx` (NEW)

A ~25-line pure presentational tile. Four instances sit inside the
`DailyDigest` grid. Takes `label`, `primary` (ReactNode), and
optional `secondary` (ReactNode). The `primary` slot is a ReactNode
— not a string — so the same shell renders big numbers, state
pills, empty-state messages, and even buttons without per-tile
variants.

### `components/ideas/DailyDigest.tsx` (NEW)

Composes the full card:

- **Time-aware greeting** — `Sunrise` / `Sun` / `Sunset` / `Moon`
  picked off `new Date().getHours()`, with "Good morning / afternoon
  / evening / Still up?" label. Memoized on mount (the user won't
  usually leave this tab open across a time-of-day boundary; if they
  do, the stale greeting is harmless).
- **Four metric tiles:**
  1. **Yesterday** — posts shipped yesterday + trend delta vs the
     day before (green/red/neutral). Empty state "0 posts · pipeline
     paused?" in zinc-500 (not alarming red, per spec).
  2. **Week fill** — `scheduledTotal / targetTotal` + mini progress
     bar. Bar color gated by percent: gold `#c5a062` below 70%,
     amber 70–99%, emerald at 100%.
  3. **Pending** — count of `pending_approval` posts; "All clear" in
     emerald when zero. `Review ▸` button routes to `pipeline` view
     (the pipeline tab hosts the approval queue; there's no
     dedicated `approval` ViewType today).
  4. **Pipeline** — state pill (Running / Paused / Off) with a dot
     (pulsing emerald / steady amber / zinc). Clicking the pill
     opens the pipeline tab; the secondary line shows
     `last: {relativeAgo}` from `pipelineLog[0].timestamp`, or
     `queue: N` if the log is empty but ideas are staged.
- **Footer row:**
  - "Next post · {relative} · {platforms}" from the earliest
    future `scheduled` post, with "Today" / "Tomorrow" / weekday
    formatting.
  - Platform strip: four dots (IG / PN / TW / DC), emerald when the
    corresponding `settings.apiKeys.<platform>` is set, zinc when
    not. This is the minimal-viable subset of the spec's health
    dot — full 4-state (healthy/stale/broken/unused) logic belongs
    to Part A.
- **Hide/show persistence** via `localStorage['mashup.ideas.digestHidden']`
  ('1' / '0'). When hidden, a `Show digest ▾` button replaces the
  card. `aria-expanded` + `aria-controls` link the toggle to the
  region.

Responsive grid: 1-col mobile → 2×2 tablet → 4-col desktop via
`grid-cols-1 sm:grid-cols-2 md:grid-cols-4`.

A11y: `role="region" aria-label="Daily digest"` on the container;
all decorative icons `aria-hidden`; trend direction lives in text
("+2 from day before") not just color.

### `components/MainContent.tsx`

Two edits:
- Import `DailyDigest`.
- Mount `<DailyDigest setView={setView} />` immediately after the
  Ideas section header's closing `</div>` and before the kanban
  columns. The spec pointed at L1908 but the actual insertion point
  in the current file is between L1941 (section header close) and
  L1943 (kanban-columns comment).

## Out of scope (deliberate)

- **Part A (platform health dot in Sidebar).** Separate component,
  separate mount site, separate task. The digest's footer platform
  strip uses a minimal inline derivation (configured vs not) since
  the full 4-state HealthDot doesn't exist yet. When Part A ships,
  the footer should switch to rendering shared HealthDots.
- **"Auto-fill" button on empty week state.** Spec calls it out as a
  "V030-DES-001 §B hook" — depends on a helper that doesn't exist
  here yet. Falls back to the standard week-fill bar + "0 / target"
  label.
- **`setView('approval')` for the Pending tile.** There is no
  `approval` ViewType in `types/mashup.ts:247`. The approval queue
  lives inside the Pipeline tab, so `Review ▸` routes to
  `setView('pipeline')`.
- **Active token validation pings.** Spec itself flags this as a
  separate PROP. The footer strip only reports configured-vs-not.
- **Reactive greeting.** The greeting computes once on mount. If
  someone leaves the Ideas tab open from 10:59am to 11:01am they'll
  see "Good morning" for longer than they should; the cost of fixing
  this (1-minute `setInterval` or a `useEffect` on focus) outweighs
  the benefit.
