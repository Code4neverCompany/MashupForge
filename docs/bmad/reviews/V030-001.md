## Outcome

- `hooks/usePipeline.ts`: **933 → 110 LOC** thin composer.
- `hooks/usePipelineDaemon.ts`: **730 LOC** — owns all pipeline state, outer cycle loop, checkpoint storage, autoGenerateIdeas, user-facing controls.
- `hooks/useIdeaProcessor.ts`: **213 LOC** — stateless, builds `ProcessIdeaDeps` bag, delegates to pure `lib/pipeline-processor.ts::processIdeaFn`.
- Public surface unchanged: `MashupContextType` consumers are untouched.
- Refs: **8 useEffect→ref sync pairs → 1** consolidated `latestPropsRef` (documented exception, spec §4.3) + **2 AbortControllers** in `useState` for stop/skip.

## Refs eliminated

| Old ref | Replacement |
|---|---|
| `stopRequestedRef` | `runCtrl: AbortController \| null` in `useState` — `signal.aborted` reads live on the shared object |
| `skipCurrentIdeaRef` | `skipCtrl: AbortController \| null` — fresh per idea |
| `pipelineContinuousRef`, `pipelineIntervalRef`, `pipelineTargetDaysRef`, `pipelineDelayRef` | Functional setState peek: `setX(prev => { captured = prev; return prev; })` |
| `checkpointImagesRef` | Local array inside `processIdea` closure, passed via deps bag |
| `checkpointSettingsRef` | Local snapshot in `runOuterLoop` closure, threaded into `writeCheckpointBase` per-call |
| `ideasRef`, `settingsRef`, `imagesRef`, `savedImagesRef` | **One** consolidated `latestPropsRef` (the documented exception — see spec §4.3) |

Previous partial refactor left `processIdea` **double-processing** every idea: lines 252-306 built deps + called `processIdeaFn`, then lines 308-619 repeated the same logic inline. The new `useIdeaProcessor` removes the duplication cleanly.
