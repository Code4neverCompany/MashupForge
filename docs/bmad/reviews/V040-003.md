# V040-003 — Single helper for verbatim caption fan-out

## Why

REFACTOR-001 + SHOULDFIX-002 already collapsed the AI-generation fan-out
into `fanCaptionToGroup`. Two inline propagation loops survived: the
captioning view's textarea `onChange` (L2570-2576) and the post-ready
view's textarea `onChange` (L3786-3792). Both did the same thing —
"copy the user's typed caption into every image in the carousel" — but
each rolled its own `for (const ci of group) patchImage(ci, { postCaption })`
loop. `fanCaptionToGroup` itself also still had its own sibling-fan-out
loop with the WARN-1 "don't overwrite manually-edited siblings" guard
inlined.

Three nearly-identical loops drift; this task collapses them into one.

## What changed

### `components/MainContent.tsx` — new helper `propagateCaptionToGroup`

```ts
const propagateCaptionToGroup = (
  group: GeneratedImage[],
  caption: string,
  hashtags: string[] | undefined,
  opts: { skipExisting?: boolean; excludeId?: string } = {},
) => { ... }
```

- `skipExisting`: when true, skips images that already carry a
  `postCaption`. Lifted from `fanCaptionToGroup`'s inline WARN-1 guard
  so the rule lives in exactly one place.
- `excludeId`: when set, skips that one image. `fanCaptionToGroup`
  uses it to avoid re-patching the anchor (which `generatePostContent`
  already wrote to).
- `hashtags === undefined` leaves each image's hashtags untouched —
  the textarea callers don't touch hashtags, so they pass `undefined`.

### `fanCaptionToGroup` (rewritten body)
The sibling fan-out loop is gone. After `generatePostContent` returns,
the helper now calls:
```ts
propagateCaptionToGroup(rest, withCaption.postCaption, withCaption.postHashtags, {
  skipExisting: !force,
  excludeId: anchor.id,
});
```

### Captioning view textarea (L2570-2573)
```diff
-for (const ci of entry.images) {
-  patchImage(ci, { postCaption: e.target.value });
-}
+propagateCaptionToGroup(entry.images, e.target.value, undefined);
```

### Post-ready view textarea (L3786-3789)
```diff
-for (const ci of item.images) {
-  patchImage(ci, { postCaption: e.target.value });
-}
+propagateCaptionToGroup(item.images, e.target.value, undefined);
```

## Out of scope (deliberate)

- **Other inline `for ... patchImage` loops** in MainContent.tsx —
  L727 (postedAt/postedTo), L737 (postError), L2621 (isPostReady=true),
  L3924 (isPostReady=false). They propagate different fields with
  different semantics; lumping them under a single "propagate
  anything" helper would lose the per-field intent. The task says
  *unify caption propagation*, not *unify all field propagation*, so
  these stay as-is.
- **Lifting `propagateCaptionToGroup` out of MainContent.tsx into
  `lib/`.** It closes over `patchImage` (which closes over
  `saveImage`), and lifting it would either need DI or a
  reorganization beyond what this task asks for. REFACTOR-001's
  follow-up note in TEST-001.md already flagged that broader
  extraction.
