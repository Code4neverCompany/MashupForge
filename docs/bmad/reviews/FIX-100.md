# FIX-100 — MainContent.tsx monolith extraction (umbrella)

**Date:** 2026-04-15
**Author:** developer (slices A + B); other agent (slices C + D)
**Status:** Slices A–D shipped. Umbrella stays open for further passes if needed.

## Problem

`components/MainContent.tsx` had grown to ~5786 LOC. Symptoms:

- **Edit-conflict surface.** Multiple agents working in parallel kept
  colliding on this one file. STORY-132 (drag-drop fix) took two
  attempts because a parallel POLISH agent scooped uncommitted edits
  and shipped them under its own commit (`52a756f`).
- **Cognitive load.** Single function holding state, effects, and
  ~10 modals with no module boundaries.
- **Tooling lag.** Tsc / eslint / IDE jump-to-symbol all slow on a
  file this large.

## Approach

Per PROP-009, extract the largest self-contained dialogs first as
independently-shippable slices. Each slice gets its own commit so
review effort is proportional to the slice, not the umbrella. No
behavioral changes — pure mechanical extraction with typed props.

## Design constraints honored

- **Pure mechanical extraction.** No prop renaming, no behavioral
  tweaks, no removed features. JSX is byte-equivalent except for
  threading state down via props.
- **Typed prop interfaces.** Each new file declares its props
  explicitly so TS catches missing wiring at the call site.
- **No new dependencies.** Each new file imports the same icons,
  context types, and lucide pieces that the inline block was using.
- **Each slice is independently revertable.** Single commit, single
  modal, single new file.

## What's NOT in scope (still in MainContent.tsx)

The remaining ~4500 LOC of MainContent are mostly tab content
(Studio / Gallery / Pipeline / Captioning), keyboard handlers,
generation pipeline state, and the smart-scheduler hookup. Those are
not "modals" and don't have the same self-contained edges that made
A–D safe extractions. Further extraction would need a real refactor
plan, not slice-and-go.

Suggested next pass if/when the file becomes painful again: extract
the **Gallery filter bar** (search + tags + collection + universe
filters) into a hook + presentational component pair. That cluster has
~150 LOC of state and ~200 LOC of JSX with a clean interface (read
filter state, write filter state, render the chips).
