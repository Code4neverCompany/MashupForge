# V060-002 — Settings Desktop tab platform toggles

**Status:** done
**Branch:** main
**Tests:** 467 → 483 (+16)

## Why

The Desktop tab listed every API field unconditionally — Leonardo,
five AI-provider keys, Instagram (2 fields), Twitter (4), Pinterest
(2), Discord (1), plus the Updates dropdown. Ten secret fields stacked
in a single panel, most of them irrelevant for any given user.

Maurice asked for a per-platform toggle so non-core platforms collapse
to a one-line row until activated. Instagram stays expanded because
the rest of the app assumes it.

## What changed

- **`lib/desktop-config-keys.ts`** — added three new env-var keys
  (`TWITTER_ENABLED`, `PINTEREST_ENABLED`, `DISCORD_ENABLED`) so the
  PATCH endpoint accepts them, and a new `PLATFORM_GROUPS` constant
  that is the single source of truth for how fields cluster by
  platform. `PLATFORM_OWNED_KEYS` lets the panel skip platform fields
  in its generic flat loop. `isPlatformEnabled` / `platformEnabledDefault`
  encode the toggle-state contract: explicit `'1'`/`'0'` wins; absent
  flag falls back to "on if any cred exists" so existing setups don't
  get hidden after the redesign.
- **`components/DesktopSettingsPanel.tsx`** — new `PlatformGroupSection`
  component (compact row + toggle + collapsible field list). The main
  render now filters platform-owned keys out of the flat field loop and
  renders a dedicated `Platforms` section that maps `PLATFORM_GROUPS`.
  Instagram is `alwaysOn` so its toggle is hidden and its fields are
  always visible. Toggles flow through the existing autosave debounce
  → PATCH `/api/desktop/config` path; no new write code.
- **`tests/lib/platform-groups.test.ts`** (new, 11 cases) — pins the
  group contract: ordering, alwaysOn for Instagram, every key declared
  in `DESKTOP_CONFIG_KEYS`, defaults flow through the toggle resolver
  in both directions.
- **`tests/integration/desktop-platform-toggle.test.tsx`** (new, 5
  cases) — pins the user-visible behavior: Instagram has no toggle and
  its fields render unconditionally, Twitter/Pinterest/Discord default
  OFF on empty config, clicking the toggle expands the group and
  PATCHes the enable flag, graceful migration keeps the platform
  expanded when stored creds exist, disabling does not wipe the keys.

## Notes

- Toggling OFF persists `'0'`, not an empty string — the API endpoint
  treats empty as a delete, and we want the explicit-off intent to
  survive across loads (otherwise the default-on-if-creds heuristic
  would re-expand the platform every time).
- The toggle is a visibility flag, not a credential wipe. The existing
  field values stay in `config.json` and round-trip through PATCH so
  the user can flip the platform back on without re-pasting tokens.
- Icons reuse the onboarding wizard convention (Camera = Instagram,
  MessageCircle = Twitter / X, Pin = Pinterest, Hash = Discord) — the
  `lucide-react` build does not ship the brand-name icons.
